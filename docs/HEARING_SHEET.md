## A. ビジネス要件

### A-1. 背景・課題
- スクショがギャラリーで埋もれて探せなくなる
- 手動フォルダ移動が地味に面倒
- 後から整理しようとしても放置しがち
- **「撮った瞬間に」フォルダを決めて振り分けたい**が本質

### A-2. ターゲットユーザー
- 副業・SNS発信者（個人利用）
- 年齢層: 【要確認】想定20〜40代
- ITリテラシー: 中程度

### A-3. 成功の定義
- 【要確認】まずは自分がほしいものを作る

### A-4. 競合
- Stockey / SnapSort / Pixel Screenshots / CamFolder
- 差別化: 共有シートからフォルダ即振り分け特化は競合なし

## B. 機能要件

### B-1. コア機能
MVP: 共有シート連携→フォルダ一覧→選択→端末保存→新規作成
将来: クラウド保存(Ph2) / モザイク(Ph3) / AI提案(Ph3)

### B-2. ユースケース
UC-1: 感想DM → スクショ → 共有 → 「お客様の声」→ 保存
UC-2: 売上画面 → スクショ → 共有 → 「実績」→ 保存
UC-3: 新カテゴリ → 共有 → 「+新規」→ 名前入力 → 保存

### B-3. データ
- 入力: スクショ画像(PNG/JPG) OS共有シート経由
- 出力: 端末内指定フォルダ
- 元スクショ削除オプション: 【要確認】

### B-4. 外部連携
- MVP: なし（端末完結）
